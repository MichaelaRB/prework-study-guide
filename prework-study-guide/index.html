HTML

  • The head element contains information about the webpage.
  • The metadata contained in the head element is not visible on the webpage.
  • The !DOCTYPE element tells the browser what type of document it should expect. !DOCTYPE html tells the webpage to expect an HTML document.
  • The html element represents the start and end of our document.The opening tag defines the language the browser will use. For example, the opening tag for an english browser would read html lang=en.
  • The title element defines the title for the webpage.
  • The style element defines associated CSS styles.
  • The base element defines the webpage's base URL, and the link element connects an external resource.
  • The meta element defines metadata, such as the available character set.
  • The script element embeds client-side scripts.
  • The body element represents the visible content shown to the user.
  • The id attribute is used as a way to identify and select elements.
  • The ul element creates an unordered list, like this one! The ol element creates a numbered list.
  • For hyperlinks, the a element is used to create a linke, and the href attribute points to the URL.
  • The img element is used to add images. The src attribute points to the location of the image, alt is a string of text that describes the image, and you can set the width and height.
  • The p element represents a block of text.
  • The h1, h2, h3, h4, h5, and h6 elements represent different levels of headings. The h1 element is the largest heading, and h6 is the smallest.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Inline CSS is applied to a specifice HTML tag, making them useful for quick and permanent changes but are a lot less flexible than style sheets.
  • An internal CSS style sheet is embedded into the document with the style element inside the head element. The CSS rules will only apply to this webpage.
  • External CSS style sheets are the ones most commonly used as they allow us to keep the rules in their own file, making it easier to make design changes.
  • You use the link element in the head element to link an external style sheet. Ex. link rel="stylesheet" href = "./assets/style.css", where rel defines the relationship and href points to the external sheet.
  • Using the * selector means that the rule will apply to every page element.
  • You can group together two or more elements as selectors if they are going to share the same styling. Ex. header, footer
  • CSS rules are structured as follows: selector {property1: value1; property2: value2; ...}
  • We can use class attributes to share a CSS rule to any element of our choosing by assigning the rule to an attribute with a selector.
  • You assign the class attribute to the element in HTML. Ex. section class="selector" id="section-id".
  • Class selectors will have a period in front of them when defining rules in the style sheet. Ex. .card.
  • When setting a value for the color property, you can use either a semantic reference such as gray or blue, or by using its hexadecimal code.

Git

  • git status: checks the current brance that we are on
  • git checkout -b branchname: creates a new branch with the specified name and moves to it.
  • git pull origin main: merges commits from the main branch into the current branch.
  • git add -A: commits all files in the working branch.
  • git commit -m "message": makes a commit, -m flag associates a message with our commit.
  • git push origin feature/branchname: pushes up changes made in the branch

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Functions are blocks of code that are not run unless called, making them reusable.
  • Arrays store multiple values at a time.
  • Arrays are zero indexed, which means that the first index of the array is always 0.
  • For loops allow a code block to be executed repeatedly.
  • JavaScript is weakly typed, meaning it allows type conversion when an operation involves mismatched types.
  • Primitive types aside from null and undefined have built in methods you can use, such as toExponential() for the Number object.
